# Data Documentation

## Overview

The Restaurant Voice Ordering Agent uses a synthetic pizza restaurant menu and project-authored FAQ/policy text. The data supports ordering, pricing, validation, and retrieval-augmented menu question answering.

## Data Sources

| Data Item | Location | Source | Version | License / Ownership |
|---|---|---|---|---|
| Restaurant menu | `data/menu/menu.json` | Project-created synthetic pizza restaurant menu | `restaurant-menu-v1` | Course project synthetic data |
| FAQ chunks | `src/restaurant_text.py` | Project-authored FAQ text | `restaurant-faq-v1` | Course project synthetic data |
| Modification policy chunks | `src/restaurant_text.py` | Project-authored customization policy text | `restaurant-policy-v1` | Course project synthetic data |
| Knowledge chunks | Generated by `src/knowledge_builder.py` | Derived from menu + FAQ constants | `restaurant-knowledge-v1` | Generated artifact |
| Chroma vector store | `data/vector_store/chroma_db` | Generated from knowledge chunks | Regenerated locally | Generated artifact, not committed |

## Menu Contents

The menu supports:

- Pizzas
- Pizza sizes
- Pizza base prices
- Crust options
- Toppings
- Sides
- Drinks
- Desserts

The default menu path is:

data/menu/menu.json

## Generated Knowledge Chunks

`src/knowledge_builder.py` converts menu records and FAQ constants into retrievable text chunks.

The generated chunks describe:

- Pizza ingredients
- Pizza prices
- Crust options and extra costs
- Topping prices
- Side, drink, and dessert options
- Dietary hints
- Allergen-related hints
- Recommendation hints
- Modification policies

## Vector Store Regeneration

The Chroma vector store is generated from source menu/FAQ data and should not be treated as source data.

Run:

python -m src.build_vector_store

Expected output:

Chroma vector store built.

## Data Limitations

- The restaurant data is synthetic.
- Prices are sample values.
- The data is not connected to a real restaurant or POS system.
- Nutrition facts such as calories, protein, sodium, and macros are not provided.
- Allergen information is limited to manually encoded hints.
- Cross-contamination safety is not guaranteed.
- The system should not infer unsupported claims from missing data.

## Data Privacy

The project does not require stored user accounts. During local testing, request inputs may appear in logs or test output. Users should not enter real payment information, addresses, or sensitive personal data.
